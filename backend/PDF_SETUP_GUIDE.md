# PDF Generation System Setup Guide

## Overview
This guide will help you set up the PDF generation system for School Report SaaS using **wkhtmltopdf** to ensure the PDF output is **100% identical** to the HTML preview.

## Prerequisites
- Python 3.8+
- Django project running
- `pdfkit` Python package (already in requirements.txt)

## Step 1: Install wkhtmltopdf

### Windows
1. Download the installer from: https://wkhtmltopdf.org/downloads.html
2. Choose: "Windows (MSVC 2015) 64-bit" 
3. Run the installer (wkhtmltox-0.12.6-1.msvc2015-win64.exe)
4. Install to default location: `C:\Program Files\wkhtmltopdf\`

**Or use the automated script:**
```bash
python install_wkhtmltopdf.py
```

### Linux (Ubuntu/Debian)
```bash
sudo apt-get update
sudo apt-get install wkhtmltopdf
```

### Linux (CentOS/RHEL)
```bash
sudo yum install wkhtmltopdf
```

### macOS
```bash
brew install wkhtmltopdf
```

## Step 2: Verify Installation

Test that wkhtmltopdf is working:
```bash
wkhtmltopdf --version
```

You should see output like:
```
wkhtmltopdf 0.12.6 (with patched qt)
```

## Step 3: Configure Django Settings

The settings are already configured in `settings.py`:

```python
# wkhtmltopdf configuration for PDF generation
if os.name == 'nt':  # Windows
    WKHTMLTOPDF_CMD = r"C:\Program Files\wkhtmltopdf\bin\wkhtmltopdf.exe"
else:  # Linux/Unix
    WKHTMLTOPDF_CMD = '/usr/local/bin/wkhtmltopdf'
```

The required packages are already in `requirements.txt`:

### Method 2: Manual Test
Create a test file `test_pdf.py`:

```python
import pdfkit
import os

# Configure for your OS
if os.name == 'nt':  # Windows
    config = pdfkit.configuration(wkhtmltopdf=r"C:\Program Files\wkhtmltopdf\bin\wkhtmltopdf.exe")
else:
    config = pdfkit.configuration()

# Test HTML
html = """
<!DOCTYPE html>
<html>
<head>
    <title>Test</title>
    <style>
        body { font-family: Arial, sans-serif; padding: 20px; }
        h1 { color: #333; }
    </style>
</head>
<body>
    <h1>PDF Test</h1>
    <p>This is a test PDF generated by wkhtmltopdf.</p>
</body>
</html>
"""

# Generate PDF
options = {
    'page-size': 'A4',
    'margin-top': '10mm',
    'margin-right': '10mm',
    'margin-bottom': '10mm',
    'margin-left': '10mm',
    'encoding': 'UTF-8',
    'quiet': ''
}

pdf = pdfkit.from_string(html, 'test.pdf', configuration=config, options=options)
print("✅ PDF generated successfully: test.pdf")
```

Run the test:
```bash
python test_pdf.py
```

## Step 6: How the System Works

### Preview System
- Uses Django view: `terminal_report_preview()`
- Renders template: `reports/terminal_report.html`
- Shows in iframe with exact styling

### PDF System  
- Uses Django view: `generate_pdf_report()`
- Uses **SAME** template: `reports/terminal_report.html`
- Uses **SAME** context function: `_get_report_context()`
- Generates PDF with wkhtmltopdf

### Key Features
✅ **Same Template**: Both preview and PDF use `terminal_report.html`
✅ **Same Context**: Both use `_get_report_context()` function  
✅ **Same Styling**: CSS optimized for both screen and PDF
✅ **Image Support**: Absolute URLs for logos and photos
✅ **Perfect Layout**: Tables, fonts, and spacing match exactly

## Step 7: API Usage

### Generate PDF Report
```javascript
// Frontend API call
const response = await fetch('/api/reports/report-cards/generate_pdf_report/', {
    method: 'POST',
    headers: {
        'Authorization': `Bearer ${token}`,
        'Content-Type': 'application/json'
    },
    body: JSON.stringify({
        student_id: 123,
        term_id: 456
    })
});

const blob = await response.blob();
const url = window.URL.createObjectURL(blob);
const a = document.createElement('a');
a.href = url;
a.download = 'student_report.pdf';
```

## Step 8: Troubleshooting

### Common Issues

**1. "wkhtmltopdf not found"**
- Ensure wkhtmltopdf is installed
- Check the path in `settings.WKHTMLTOPDF_CMD`
- Verify with: `wkhtmltopdf --version`

**2. "Images not showing in PDF"**
- Check that `MEDIA_URL_BASE` is set correctly in settings
- Ensure image files exist and are accessible
- The system automatically converts to absolute URLs

**3. "PDF layout different from preview"**
- The system uses identical templates and context
- Check browser console for CSS errors
- Ensure all CSS is inline or properly linked

**4. "Permission denied"**
- On Linux: `sudo chmod +x /usr/local/bin/wkhtmltopdf`
- Check file permissions for media files

### Debug Mode
Enable debug logging in `settings.py`:
```python
LOGGING = {
    'version': 1,
    'handlers': {
        'console': {
            'class': 'logging.StreamHandler',
        },
    },
    'loggers': {
        'reports.pdf_generator': {
            'handlers': ['console'],
            'level': 'DEBUG',
        },
    },
}
```
