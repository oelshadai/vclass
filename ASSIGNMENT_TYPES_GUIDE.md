# Assignment Types - Complete Flow Guide

## 📚 Overview
This guide explains how each assignment type works in the system, from creation to student submission.

---

## 1️⃣ HOMEWORK 📝

### **Purpose**
Regular assignments where students submit written work or files.

### **Creation Flow**
1. Click "New Assignment"
2. Select Type: **Homework**
3. Fill in:
   - Title (e.g., "Math Chapter 5 Problems")
   - Description
   - Instructions (detailed)
   - Class
   - Due Date
   - Max Score
   - Max Attempts (default: 1, can increase)
4. Click "Create & Publish"
5. ✅ **Published immediately** - Students can start working

### **Features**
- ✅ File submissions allowed
- ✅ Text submissions allowed
- ✅ Multiple attempts (configurable)
- ✅ Manual grading by teacher
- ❌ No time limit
- ❌ No auto-grading

### **Student Experience**
1. See assignment in their dashboard
2. Upload files or write text
3. Submit before due date
4. Can resubmit if attempts remain
5. Wait for teacher grading

### **Teacher Grading**
1. View submissions
2. Download/review student work
3. Assign score
4. Provide feedback
5. Mark as graded

---

## 2️⃣ QUIZ ⏱️

### **Purpose**
Timed assessments with multiple-choice or short-answer questions. Auto-graded.

### **Creation Flow**
1. Click "New Assignment"
2. Select Type: **Quiz**
3. Fill in:
   - Title (e.g., "Week 3 Math Quiz")
   - Description
   - Instructions
   - Class
   - Due Date
   - Max Score
   - **Time Limit** (required, e.g., 30 minutes)
   - Max Attempts (default: 3)
4. Click "Next: Add Questions"
5. **Add Questions:**
   - Enter question text
   - Select type (MCQ or Short Answer)
   - Set points
   - For MCQ: Add options and mark correct answer
   - Click "Add Question"
   - Repeat for all questions
6. Click "Publish Assignment"
7. ✅ **Published with questions** - Students can take quiz

### **Features**
- ✅ Timed (countdown timer)
- ✅ Auto-graded
- ✅ Multiple attempts (default: 3)
- ✅ Immediate results
- ✅ Question bank
- ❌ No file uploads

### **Student Experience**
1. See quiz in dashboard
2. Click "Start Quiz"
3. Timer starts automatically
4. Answer all questions
5. Submit before time runs out
6. See score immediately
7. Can retake if attempts remain

### **Validation**
- ⚠️ Must have at least 1 question
- ⚠️ Must have time limit
- ⚠️ Total question points should match max score

## 3️⃣ PROJECT 🎯

### **Purpose**
Long-term assignments requiring detailed work, research, or practical implementation.

### **Creation Flow**
1. Click "New Assignment"
2. Select Type: **Project**
3. Fill in:
   - Title (e.g., "Science Fair Project")
   - Description
   - Instructions (very detailed)
   - Class
   - Due Date (typically weeks away)
   - Max Score
   - Max Attempts (default: 1)
4. Click "Create Project (Draft)"
5. ✅ **Saved as DRAFT** - Not visible to students yet
6. Edit assignment to add:
   - Detailed requirements
   - Rubric
   - Milestones
   - Resources
7. Click "Publish" from assignment list
8. ✅ **Now published** - Students can see and start

### **Features**
- ✅ File submissions required
- ✅ Text submissions allowed
- ✅ Long due dates
- ✅ Manual grading
- ✅ Detailed instructions
- ❌ No time limit
- ❌ No auto-grading

### **Student Experience**
1. See project in dashboard
2. Read detailed requirements
3. Work over extended period
4. Upload project files
5. Submit before deadline
6. Wait for teacher evaluation

### **Teacher Grading**
1. Download project files
2. Review against rubric
3. Assign score
4. Provide detailed feedback
5. Mark as graded

## 4️⃣ EXAM 📋

### **Purpose**
Formal timed assessments. Most restrictive type. Single attempt only.

### **Creation Flow**
1. Click "New Assignment"
2. Select Type: **Exam**
3. Fill in:
   - Title (e.g., "Midterm Exam - Mathematics")
   - Description
   - Instructions (exam rules)
   - Class
   - Due Date
   - Max Score
   - **Time Limit** (required, e.g., 90 minutes)
   - Max Attempts (locked at 1)
4. Click "Next: Add Questions"
5. **Add Questions:**
   - Enter question text
   - Select type (MCQ or Short Answer)
   - Set points
   - For MCQ: Add options and mark correct answer
   - Click "Add Question"
   - Repeat for all questions
6. Click "Publish Assignment"
7. ✅ **Published with questions** - Students can take exam

### **Features**
- ✅ Timed (strict countdown)
- ✅ Auto-graded
- ✅ Single attempt only (cannot retake)
- ✅ Immediate results
- ✅ Question bank
- ⚠️ Most restrictive
- ❌ No file uploads

### **Student Experience**
1. See exam in dashboard
2. Click "Start Exam" (one time only!)
3. Timer starts - cannot pause
4. Answer all questions
5. Submit before time runs out
6. See score immediately
7. ❌ Cannot retake

### **Validation**
- ⚠️ Must have at least 1 question
- ⚠️ Must have time limit
- ⚠️ Max attempts locked at 1
- ⚠️ Total question points should match max score

## 🎯 Best Practices

### **HOMEWORK**
- ✅ Clear, detailed instructions
- ✅ Reasonable due dates
- ✅ Specify file formats accepted
- ✅ Provide rubric

### **QUIZ**
- ✅ 10-20 questions typical
- ✅ Mix question types
- ✅ Allow 2-3 attempts
- ✅ 30-45 minute time limit
- ✅ Test before publishing

### **PROJECT**
- ✅ Very detailed requirements
- ✅ Provide examples
- ✅ Set milestones
- ✅ Long due dates (2-4 weeks)
- ✅ Clear grading rubric

### **EXAM**
- ✅ 20-50 questions typical
- ✅ Comprehensive coverage
- ✅ Single attempt only
- ✅ 60-120 minute time limit
- ✅ Test thoroughly before publishing
- ✅ Announce exam date in advance

### **EXERCISE**
- ✅ Quick and focused
- ✅ Practice-oriented
- ✅ Allow multiple attempts
- ✅ Fast feedback

## 🚨 Common Issues & Solutions

### **Issue: "Quiz/Exam must have questions"**
**Solution:** Add at least one question before publishing

### **Issue: "Time limit required"**
**Solution:** Set time limit for Quiz/Exam types

### **Issue: "Cannot publish project immediately"**
**Solution:** Projects save as draft - add details then publish from list

### **Issue: "Students can't see assignment"**
**Solution:** Check assignment status is PUBLISHED, not DRAFT

### **Issue: "Question points don't match max score"**
**Solution:** Adjust question points or max score to match
